# 收盘价时间序列预测 - 模型预测逻辑详解

## 概述

本系统实现了 **5种简单预测模型** + **3种高级预测模型** 的组合预测策略，通过加权融合的方式提高预测准确性。所有模型均采用滚动预测（Rolling Forecast）方式对测试集进行逐点预测。

---

## 一、五种简单预测模型

### 1. 朴素模型 (Naive Model)

**预测公式：**
```
ŷ(t+h) = y(t)
```

**预测逻辑：**
- 假设未来的值等于最后一个观测值
- 所有预测步长都使用相同的值（最后观测值）

**特点：**
- 最简单的基准模型
- 适用于平稳序列、随机游走过程
- 对短期预测效果较好

**适用场景：** 没有明显趋势和季节性的数据

---

### 2. 季节性朴素模型 (Seasonal Naive Model)

**预测公式：**
```
ŷ(t+h) = y(t+h-m×k)
```
其中：
- `m` = 季节周期（默认7天）
- `k` = 使得 `t+h-m×k` 落在历史数据范围内的整数

**预测逻辑：**
- 利用数据的季节性模式
- 预测未来某个时刻的值等于上一个周期相同时刻的观测值
- 例如：预测下周一使用本周一的值，预测下周二使用本周二的值

**特点：**
- 捕捉周期性模式
- 对强季节性数据效果好

**适用场景：** 具有明显周周期、月周期等季节性特征的数据

---

### 3. 均值模型 (Mean Model)

**预测公式：**
```
ŷ(t+h) = (1/n) × Σy(i),  i=1 to n
```

**预测逻辑：**
- 计算所有历史数据的算术平均值
- 所有未来时刻的预测值都相同（全局均值）

**特点：**
- 假设数据围绕稳定均值波动
- 忽略趋势和季节性
- 对异常值敏感

**适用场景：** 平稳序列、无明显趋势或季节性的数据

---

### 4. 漂移模型 (Drift Model)

**预测公式：**
```
ŷ(t+h) = y(t) + h × [(y(n) - y(1)) / (n-1)]
```

**预测逻辑：**
- 在朴素模型基础上增加线性趋势项（漂移）
- 漂移率 = (最后一个观测值 - 第一个观测值) / (总样本数 - 1)
- 假设数据以恒定速率变化

**特点：**
- 捕捉线性趋势
- 外推趋势可能不准确（长期预测）
- 对首尾观测值敏感

**适用场景：** 具有稳定线性趋势的数据

---

### 5. 动态均值模型 (Dynamic Mean / Rolling Mean)

**预测公式：**
```
ŷ(t+h) = (1/k) × Σy(i),  i=t-k+1 to t
```
其中：
- `k` = 滚动窗口大小（默认5）

**预测逻辑：**
- 使用最近k个观测值的平均值
- 窗口随时间滑动，始终使用最新数据
- 相比全局均值更能捕捉近期变化

**特点：**
- 对数据变化更敏感
- 窗口大小影响平滑程度
- 小窗口：灵敏但噪声大；大窗口：平滑但滞后

**适用场景：** 局部平稳、短期波动的数据

## 二、三种高级预测模型

### 1. AR(p) 自回归模型 (AutoRegressive Model)

**预测公式：**
```
ŷ(t) = c + φ₁×y(t-1) + φ₂×y(t-2) + ... + φₚ×y(t-p) + ε(t)
```

**核心算法：**

#### 阶数选择（PACF方法）
1. 计算偏自相关函数（PACF）
   - 使用 Durbin-Levinson 算法递归计算
   - 最大搜索阶数：30

2. 显著性检验
   - 95%置信区间边界：±1.96/√n
   - 选择最后一个超出显著性边界的滞后阶数作为p

3. 阶数限制
   - 最小阶数：1
   - 最大阶数：20

#### 参数估计
- 构建多元线性回归模型
- 使用高斯消元法求解最小二乘问题
- 估计截距项c和自回归系数φ₁, φ₂, ..., φₚ

**预测过程：**
- 单步预测：使用最后p个观测值
- 多步预测：递归使用之前的预测值

**特点：**
- 自动适应数据的自相关结构
- 适合短期预测
- 对非平稳数据效果差

**适用场景：** 自相关性强的平稳时间序列

---

### 2. STL分解外推 (Seasonal-Trend Decomposition)

**分解公式：**
```
y(t) = Trend(t) + Seasonal(t) + Residual(t)
```

**分解步骤：**

#### Step 1: 提取趋势分量
- 使用中心移动平均
- 窗口大小 = 季节周期m
- 边界值用最接近的有效趋势值填充

#### Step 2: 去趋势
```
Detrended(t) = y(t) - Trend(t)
```

#### Step 3: 提取季节分量
- 对每个季节位置（0到m-1），计算完整周期的平均值
- 例如：所有周一的平均值、所有周二的平均值...

#### Step 4: 残差分量
```
Residual(t) = y(t) - Trend(t) - Seasonal(t)
```

**预测方法：**

1. **趋势外推**
   - 使用最后20个趋势值进行线性回归
   - 外推得到未来趋势值

2. **季节分量**
   - 使用最后一个完整周期的对应位置值
   - 例如：预测第h天，使用季节分量[h mod m]

3. **合成预测**
   ```
   ŷ(t+h) = Trend_forecast(t+h) + Seasonal(h mod m)
   ```

**特点：**
- 显式建模趋势和季节性
- 可解释性强
- 需要足够长的历史数据

**适用场景：** 有明显趋势和季节性成分的数据

---

### 3. 动态滚动均值 (Dynamic Rolling Mean)

**预测公式：**
```
ŷ(t+1) = (1/k) × Σy(i),  i=t-k+1 to t
```
其中：
- `k` = 窗口大小（默认20，可在界面调整）

**预测逻辑：**
- 与简单模型中的动态均值类似
- 但在此处作为独立的高级方法参与组合
- 使用较大的默认窗口（20 vs 5）以获得更平滑的预测

**特点：**
- 平滑噪声
- 对突变响应较慢
- 计算简单高效

**适用场景：** 噪声较大、需要平滑处理的数据

## 三、组合预测策略

### 整体架构

```
最终预测 = 0.7 × 高级模型均值 + 0.3 × 简单模型均值
```

### 高级模型组合（权重70%）

**组成：**
- AR(p) 模型（1/3权重）
- STL分解模型（1/3权重）
- 动态滚动均值（1/3权重）

**计算公式：**
```
Advanced_Mean = (AR_pred + STL_pred + MA_pred) / 3
```

**设计理念：**
- 三种方法从不同角度建模
- AR捕捉自相关性
- STL捕捉趋势和季节性
- MA提供平滑基准
- 等权重平均降低单一模型风险

### 简单模型组合（权重30%）

**可选模型：**
- 朴素模型
- 季节性朴素模型
- 均值模型
- 漂移模型
- 动态均值模型

**用户选择：**
- 用户可在界面勾选任意数量的简单模型
- 选中的模型等权重平均

**计算公式：**
```
Simple_Mean = Σ(selected_model_preds) / num_selected_models
```

### 加权融合

**最终预测：**
```
Final_Prediction = 0.7 × Advanced_Mean + 0.3 × Simple_Mean
```

**权重选择理由：**
- **70%高级模型**：给予复杂模型更高权重，因为它们能捕捉更多数据模式
- **30%简单模型**：保留简单模型的稳定性，防止过拟合
- 平衡了模型复杂度和鲁棒性

## 四、滚动预测机制

### 工作原理

对于测试集的每个时间点t：

1. **更新训练集**
   ```
   Current_Training = Original_Training + Testing_Data[0:t-1]
   ```

2. **重新拟合模型**
   - 基于当前训练集重新训练所有模型
   - AR模型重新选择阶数和估计参数
   - STL重新分解趋势和季节分量

3. **单步预测**
   - 每个模型预测下一个时间点
   - 组合得到最终预测值

4. **移动到下一点**
   - 将真实值加入训练集
   - 重复步骤1-3

### 优势

- **避免信息泄露**：每次预测只使用历史信息
- **模拟真实场景**：与实际应用一致
- **准确评估性能**：反映模型在实际部署时的表现

## 五、评估指标

### 1. RMSE (均方根误差)
```
RMSE = √[(1/n) × Σ(actual_i - predicted_i)²]
```
- 对大误差惩罚更重
- 单位与原始数据相同

### 2. MAE (平均绝对误差)
```
MAE = (1/n) × Σ|actual_i - predicted_i|
```
- 对所有误差同等对待
- 鲁棒性强于RMSE

### 3. MAPE (平均绝对百分比误差)
```
MAPE = (100%/n) × Σ|(actual_i - predicted_i) / actual_i|
```
- 相对误差，便于跨数据集比较
- 不适用于实际值为0的情况

| 参数 | 默认值 | 范围 | 说明 |

## 七、使用建议

### 数据要求

1. **最小样本量**：至少50个观测值
2. **数据质量**：无缺失值、无异常值
3. **时间间隔**：等间隔时间序列

### 参数调优

1. **季节性周期**
   - 日数据：m=7（周周期）
   - 小时数据：m=24（日周期）
   - 月数据：m=12（年周期）

2. **动态窗口**
   - 短期波动大：减小窗口（3-5）
   - 长期趋势明显：增大窗口（10-20）

3. **模型选择**
   - 有季节性：必选季节性朴素模型
   - 有趋势：必选漂移模型
   - 平稳数据：均值模型效果好

### 结果解读

- **RMSE < 5%**：优秀
- **RMSE 5-10%**：良好
- **RMSE 10-20%**：一般
- **RMSE > 20%**：较差，需调整模型或参数

## 九、局限性与改进方向

### 当前局限

1. **未考虑外部因素**：仅使用历史价格
2. **线性假设**：AR和STL都是线性模型
3. **固定权重**：组合权重未自适应调整
4. **单变量**：未利用成交量等其他指标

### 改进方向

1. **引入机器学习**：LSTM、XGBoost等非线性模型
2. **自适应权重**：根据历史表现动态调整权重
3. **多变量输入**：结合成交量、技术指标
4. **集成学习**：Bagging、Boosting等方法
5. **贝叶斯方法**：提供预测不确定性区间
